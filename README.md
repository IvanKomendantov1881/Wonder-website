# ЖК WONDER — лендинг

Многостраничный лендинг жилого комплекса «WONDER»: статичная вёрстка на HTML/CSS/JS без сборщиков и бэкенда.

## Страницы

| Файл | Назначение |
|---|---|
| `index.html` | Главная: hero-блок с видео, блок «О компании», каталог, новости |
| `brifing.html` | Форма заявки/брифинга |
| `contacts.html` | Контакты (с виджетом часов) |
| `merch.html` | Мерч ЖК |

## Стек

- HTML5 / CSS3
- Vanilla JS + [jQuery](https://jquery.com/) (анимации навигации при скролле)
- [Swiper](https://swiperjs.com/) — слайдер (подключён через CDN)
- Никакой сборки — открывается прямо в браузере

## Запуск локально

Никакого сервера не требуется — файлы можно открыть напрямую в браузере:

```bash
git clone <ссылка-на-репозиторий>
cd <папка-репозитория>
open index.html   # или просто дважды кликнуть по файлу
```

Либо через локальный сервер (удобнее, если браузер ругается на CORS для видео/шрифтов):

```bash
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

## Деплой на GitHub Pages

1. Запушить репозиторий на GitHub.
2. Settings → Pages → Source: ветка `main`, папка `/ (root)`.
3. Сайт появится по адресу `https://<username>.github.io/<repo>/`.

## Что было поправлено при подготовке к публикации

Исходники были свёрстаны в Windows, где пути к файлам регистронезависимые и `\` работает как разделитель — на GitHub Pages (Linux, регистрозависимая ФС) так не будет. Поэтому:

- все пути вида `Img\file.jpg` заменены на `Img/file.jpg`;
- поправлены расхождения в регистре папок (`img\` → `Img/`, `css\` → `Css/`, `js\` → `Js/`);
- главный файл переименован из `Index.html` в `index.html` — иначе GitHub Pages не подхватит его как страницу по умолчанию.

Сама вёрстка и контент не менялись.

## Лицензия на медиа

В `Img/` есть фоновое видео `motionplaces.com-0044-c022.mp4` — судя по названию, это сток-видео с motionplaces.com. Если репозиторий публичный и сайт используется не только как учебный пример, стоит проверить лицензию на это видео и на остальные стоковые фото перед коммерческим использованием.
